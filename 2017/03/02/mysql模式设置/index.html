mysql模式设置

2017-03-02

今天错误的运用了groupby函数,导致了一个报错,通过查询虽然没有弄清楚底层的原因,不过了解到了mysql的模式问题.

Expression #1 of SELECT list is not in GROUP BY clause and contains nonaggregated column ‘ggyun.T0.id’ which is not functionally dependent on columns in GROUP BY clause; this is incompatible with sql_mode=only_full_group_by

设置的模式换一个就好了,具体原因是?
解决:在/etc/mysql/my.cnf中添加

[mysqld]
sql_mode = “STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION”
