git diff格式

2017-02-28

  • git diff 是一种常用的git指令指出文件变更的具体内容,然而每次输出的结果总是看得有些乱,很多符号不明白其中的意思.比如输入:

$ git diff

diff –git

代表git格式的diff比较命令,还有-c上下文格式 -u合并格式 等diff比较命令,具体的区别可以参考

a/controllers/BaseController.go b/controllers/BaseController.go

是第一个文件(以a表示)比较的路径和第二个文件(以b表示)的路径

index 461b0ca..4f0e859 100755

是代表暂存区文件的哈希值 461b0ca 和工作目录的文件的哈希值4f0e859,100755代表的是两文件的权限(普通文件 755权限文件)

— a/controllers/BaseController.go
+++ b/controllers/BaseController.go

变更的地方a文件以---表示,b文件以+++表示

@@ -66,11 +66,11 @@

-表示第一个文件（即啊），”66”表示第66行，”11”表示连续11行。合在一起，就表示下面是第一个文件从第66行开始的连续11行.+表示的亦如此

+this.CheckMessage()
+this.SetMessage()
this.SetUnReadNum()
this.Data[“Words”] = this.Words
this.Data[“UserName”] = this.UserName
-beego.Debug(this.UserName)
-beego.Debug(this.UserId)

+后面表示比较所得a文件的变动.-后面表示比较所得b文件的变动
