session共享机制

2017-01-09

  • 学习了nginx之后可知nginx可以作为反向代理服务器，用于负载均衡。在实际项目中用户对同一网址的访问可能在后端多个web服务器中选择一个进行处理，或这在url分发（针对特定的url指定特定的服务器处理）的情况下，如果需要用户认证等操作则会涉及到session共享的问题，现在总结如下：

不使用session，换作cookie

  • 这种方法实际上将用户身份的凭证放在了客户端，这样很容易被人伪造身份，虽然可以考虑通过加密手段将用户的身份加密记录，但终归不是正统的解决方案

ip_hash

  • 这种方案是利用nginx的ip_hash的技术将某个ip的请求定向到同一后台，但是对于url分发的场景中不适合实现。再有也无法达到负载均衡的目的了

建立redis集群或者memcached集群

  • 该方法利用集群自带的同步方法帮助我们在不同的主机中同步session，相当于把原来的一份session变成了N份session是一种不错的解决方式

session文件储存位置共享

  • 一般session都是存在内存中，可以考虑把session储存在本地磁盘中，对于虚拟主机都在同一电脑上的场景来说可以让多个虚拟主机共同使用同一session文件夹，这样就达到了session共享的目的，本次beego框架中解决方案也正是使用的这种方法
